User Guide

1. Quick Start

Install the app from the Microsoft Store and launch it. On first launch, a quick configuration wizard walks you through the essential settings (network ports, capture preferences). Accept the defaults to get started — you can change everything later from the Settings menu.

Once configured, the app starts listening for telemetry automatically. Just start the game and start driving — the app detects each session, records your telemetry, and saves it when you're done.

The main screen shows a status bar confirming the app is active. Press the indicated keys to open the menu for analysis, settings, and more.

2. Setting Up F1 25 and Your Sim Racing Software

The app sits between your game and your sim racing software (e.g., Simagic SimPro Manager, Simhub, or similar). The game sends telemetry data to the app, and the app forwards it to your sim software — so your wheel display, dashboard, and other peripherals continue to work normally.

In-game setup (F1 25)

  1. Open the Settings menu in F1 25, then find the Telemetry settings
  2. Set UDP Telemetry to On
  3. Set UDP Port to 20774 (the app's default listen port)
  4. Set UDP Send Rate to 60Hz for best accuracy
  5. Set UDP Format to 2025 (the current telemetry format used by F1 25)

Sim racing software setup

In your sim racing software, set the telemetry input port to 20775 (the app's default forward port). The app receives on 20774 and forwards to 20775 — your software reads from 20775 as if the game were sending directly.

If your software already uses a different port, consider changing it to 20775 or change the app's forward port in Settings to match.

If you don't use sim racing software

The app works fine on its own — it captures and analyzes telemetry regardless of whether forwarding is active. The forward port setting can be left default.

3. Capture Your First Telemetry

Start the app before launching a session in F1 25. The app auto-detects when a session begins and starts recording. When the session ends (or you exit), it saves the data automatically.

After your session, you'll find output files in your telemetry_logs/ folder:

  • {Date}_{Track}_{Session}_laps.csv — one row per completed lap
  • {Date}_{Track}_{Session}_turns.csv — one row per turn per lap (requires calibration data for the track)

The app refers to these files as processed telemetry files or CSV telemetry files.

Raw telemetry capture (recommended)

By default, the app also saves a raw binary capture file in telemetry_logs/raw_telemetry/. This is the complete telemetry stream from your session.

Why keep raw captures: These let you generate a specific subset of high-resolution lap charts (5-panel telemetry charts with brake/throttle, steering and lateral G, speed, gear shifts),

but more importantly allow you to replay sessions to re-process data (useful when track calibration data improves or new features are added), and create or refine track calibration files.

Every other chart in the chart analysis will still work without these, including the feature to bundle the data for third-party LLM analysis.

Disk space: A raw capture is roughly 0.5 GB for a 15-lap race. If disk space is tight, you can disable raw capture in Settings — but you'll lose the ability to replay sessions later or to generate high-resolution lap charts.

4. Review Your Telemetry Data

From the menu, select Open telemetry_logs folder to browse your output files.

What to look for in the laps CSV

Open a _laps.csv file in any spreadsheet app (Excel, Google Sheets) for a quick at-a-glance review of your session. The most useful columns to scan:

  • lap_time / sector times — your pace progression across the session
  • session type, track info — track and whether this was a FP/Quali/race session
  • ai_difficulty — the AI level you raced against
  • position — where you finished each lap
  • tyre_compound / tyre_age_laps — stint strategy and tyre age
  • assists_any — whether any assists were active (you can find the exact assists that were used in further columns)
  • damage_any — whether you picked up damage (you can find the exact damage and severity in further columns)
  • weather and time of day — session conditions
  • lap type — whether this was an inlap, outlap, safety car lap, standing start lap etc.
  • flashback used/enabled — whether flashbacks were enabled and whether they were actually used on a particular lap.
  • lockups_count / spins_count — how clean your driving was
  • traffic conditions: whether you were stuck in traffic for a particular lap, whether DRS was used, whether the car behind had DRS activated,
  • track_limits_violations — rule infringements

See CSV_OUTPUT_FORMAT.md for the complete field reference.

The turns CSV

The _turns.csv contains per-turn detail: braking points, apex speeds, throttle commitment, racing line coordinates, tyre temperatures, and more. This data is dense — it's most useful when consumed through the analysis charts rather than viewed directly.

A few fields worth checking directly:

  • position_entry / position_exit — position changes through corners (useful for reviewing race battles)
  • time_at_entry_ms on the RaceStart row — your standing start reaction time in milliseconds
  • basic setup per turn - differential, brake bias.
  • max throttle, max brake - throttle and brake profile through the turn.
  • lockups and spins severity per axle - lockup and wheelspin profile through the turn.

5. Generate Analysis Charts

From the menu, select Generate analysis charts. This creates a full set of performance charts from your session data.

Quick start

  1. The app shows your available tracks and sessions — pick one
  2. Optionally pick a second session on the same track for a comparison analysis (recommended)
  3. Press Enter to accept defaults for focus turns and lap range (the app auto-selects clean laps and the most insightful turns based on your driving style within that particular session)
  4. Charts are generated and saved to telemetry_logs/analysis/
  5. Choose to open the output folder when prompted

Single-race vs. comparison mode

  • Single race: Analyzes one session — pace consistency, technique per turn, tyre and energy management
  • Comparison (two sessions on the same track): Adds cross-race delta charts showing exactly where you gained or lost time between sessions. This is the most powerful mode for tracking your progression.

Menu options explained

  • Highlighted focus turns: The turns that get detailed technique charts (brake traces, throttle traces, speed, racing lines). Auto-detection picks the turns where you lose the most time. Override with e.g. 1,5,6+7 to focus on specific turns (use + for chicanes treated as one complex).
  • Laps to include: Which laps to analyze. Auto-select excludes inlaps, outlaps, standing start laps, and safety car laps so only representative laps are analyzed.

Override with e.g. 2-14 to include specific laps. Even when excluded, inlaps, outlaps, damage laps, standing start laps, safety car laps, are shown and analyzed in separate charts.

Highest-value charts

  • Time left on table — Shows how much time you're losing per turn compared to your own best. The biggest bars are your highest-priority improvement areas.
  • Turn time delta (comparison mode) — Waterfall chart showing where you gained or lost time between two sessions, turn by turn.
  • Braking/throttle consistency — Scatter plots of your braking point and throttle commitment distances. Tight clusters mean consistent execution; wide spread means you're losing time to variability.
  • Standing start analysis (Race/Sprint) — Reaction time, traction, throttle application, and position changes off the line.
  • Lap time progression, lap times per stint, fuel-normalized lap time - shows your lap time medians, slowest/fastest per stint, and normalizes lap time with fuel load (especially useful when comparing raw pace between 2 sessions that had different number of total laps (e.g. Medium vs Long races)).
  • Racing line (per focus turn) — Spatial view of your line through the turn, with all clean laps overlaid. Shows apex consistency and exit trajectory.
  • Technique traces (per focus turn) — Overlaid brake, throttle, speed, and gear traces for all clean laps. Shows exactly how your inputs vary lap-to-lap.
  • Scorecard — A summary card with session context (AI difficulty, assists, weather, positions). Note: the technique and consistency scores on this chart are experimental and may not be accurate — use the dedicated technique charts to confirm. Can be used for an at-a-glance overview of your strengths and opportunities within the session.

The full chart set also includes: tyre degradation vs. tyre age, ERS deployment patterns, battery lifecycle, tyre temperature window scoring, full-lap telemetry overlays (fastest vs. slowest), and advanced yaw rate / longitudinal G traces per turn.

6. Generate a Per-Lap Telemetry Plot

From the menu, select Generate a telemetry plot. This produces a high-resolution 6-panel chart for a specific lap, showing the full telemetry trace across the lap distance:

  • Brake and throttle inputs
  • Steering angle
  • Lateral G-force
  • Speed (km/h)
  • Gear
  • Turn zones (colored bands from calibration data)

This is useful for studying a single lap in detail — examining braking points, throttle application, steering smoothness, and how each turn connects to the next.

Requires a raw capture file (.bin). The menu prompts you to select a capture file, then choose the car (your car or any AI position), lap number, chart theme (dark/light), and optionally a track map overlay.

Charts are saved to telemetry_logs/charts/.

7. Generate an LLM Analysis Request

From the menu, select Prepare LLM analysis request. This generates all the analysis charts and bundles them with your CSV data and an analysis skill file into a single ZIP file, ready to upload to an AI for a coaching report. The skill file is specifically designed to extract repeatable, consistent, high-value insights and reports from the LLM, by providing it a precise framework for analysis and how to evaluate and explain it step by step, together with accurate, machine-consumable telemetry analysis (the app pre-generates and prepares the relevant telemetry analysis and charts, including metadata that helps LLMs precisely read the data).

The embedded LLM skill is designed to ensure that the LLMs produce meaningful advice and analysis, with the most accurate in-depth understanding of the data as possible.

Note The app does not use any LLM and does not connect or send data to any LLM. The prepared zip file is a bundle that can be uploaded by users to their preferred LLM (or the LLMs suggested by the app, since those have been tested to work with the analysis skill). Interactions with the LLM happen outside the scope of the app and are subject to the particular LLM terms of service.

Quick start

  1. Pick your track and session(s) — same flow as chart generation
  2. Accept defaults or customize focus turns and lap range
  3. The app creates a ZIP in telemetry_logs/llm_analysis_requests/
  4. Open the folder when prompted

How to use the ZIP

  1. Open Claude Opus 4.6, ChatGPT 5.4 Thinking, or your preferred web-based LLM
  2. Drag and drop the ZIP file into the chat
  3. Ask: "Please follow the instructions in the zip file to help me analyze my F1 telemetry, thanks!"
  4. Review the analysis, then follow up with: "Can you show me charts from the zip file that support the key priorities you've identified?"

Tip for ChatGPT users: Use stronger wording to get the full structured analysis: "Please follow the instructions in the zip file to analyze my F1 telemetry. Please systematically go through the thinking phase and output phase exactly as instructed to produce the precise intended structure, thanks!"

The ZIP contains:

  • llm_telemetry_analysis_request.md — the analysis prompt (the AI reads this first)
  • charts/ — all generated chart PNGs plus summary and manifest
  • data/ — laps and turns CSVs plus field documentation

Providing additional context

You can also provide the LLM additional context (e.g. It was the first time I raced without assists at this track. Braking felt off at a few key corners. or In the first race, I was just getting oriented on this new track so I was using racing line, in the second race I disabled racing line and I was pushing hard to find the limits of grip, therefore I crashed out on Lap 19.), or ask it for specific follow-ups: I'm not comfortable with my braking point at turn T3, how does it look from the data, and what should I focus on to improve my confidence at that turn?).

8. Replay a Telemetry File

The app refers to re-generating CSV data as replaying a telemetry file.

From the menu, select Replay a telemetry session. This re-processes a raw capture file to regenerate the CSV output — useful when calibration data has been updated or after an app update adds new features or fixes.

You can:

  • Replay all files missing output — the app finds capture files that don't have corresponding CSVs and replays them in batch
  • Select an individual file — pick a specific capture to replay

Replayed output files are prefixed with replay_ (e.g., replay_2026_01_Jan_30_AbuDhabi_Race_laps.csv).

9. Changing Listener Settings

From the menu, select Listener settings to adjust:

  • Listen port — the port the game sends telemetry to (default: 20774). Change this if the game is configured to send on a different port.
  • Forward port — the port your sim software reads from (default: 20775). Match this to your software's input port.
  • Enable telemetry capture — toggle raw binary capture on/off. On by default (recommended).
  • Enable dashboard patching — replaces the in-game fuel display with gap-behind data and brake temps with tyre wear on your wheel display. Off by default.
  • Allow broadcast IP — enable if you run the game on a different machine than the app. Off by default (localhost only).
  • Desktop shortcut — create a desktop shortcut for quick access.

After saving, the app restarts automatically with the new settings.

10. Calibration Files

Calibration files define the turn zones for each track — where each turn starts, where the apex is, and where it ends. The app uses these to break down your laps into per-turn telemetry.

What ships with the app

The app includes calibration data for the following tracks:

  • Melbourne (Australia)
  • Shanghai (China)
  • Suzuka (Japan)
  • Sakhir (Bahrain)
  • Yas Marina (Abu Dhabi)

More tracks are added with each release. These files should work out of the box — no setup needed.

For tracks without calibration data, the app still records lap-level telemetry (lap times, tyre data, etc.) but turn-by-turn analysis is not available.

Advanced: Refine existing calibration for supported tracks

If you notice turn boundaries that feel slightly off (e.g., a turn zone that starts too early or too late), you can adjust them:

Prerequisite You will have to capture a default Very short 5-lap Race on that track using an AI level of between 90 and 100. Allow the AI to finish the race, or at the minimum to complete 2 laps.

  1. From the menu, select Advanced options > Review/edit track calibration
  2. Pick the track and the AI 90 capture file.
  3. The review UI opens showing the track map with turn zones marked
  4. Click a turn to select it, drag boundary lines to adjust zone start/end
  5. Use the toolbar to remove turns, add new ones, or undo changes
  6. Confirm to save, or reject to discard

A detailed calibration editing guide will be added in a future update.

Advanced: Create calibration data for new tracks

To create calibration for a track not yet supported:

  1. In F1 25, start a very short 5-lap Race session on the target track with AI at 90-100 difficulty
  2. Make sure raw telemetry capture is enabled in the app's settings
  3. Let the AI in P1 complete 3-5 clean laps, then exit the session
  4. From the menu, select Advanced options > Create new track calibration
  5. Pick the track and the capture file you just recorded
  6. The app analyzes the telemetry to detect turn zones automatically
  7. Review and adjust the detected turns in the calibration UI, starting from the track map and progressing to the detailed 6-panel high-resolution telemetry chart
  8. Turns can be selected by clicking, and edited by dragging their boundaries
  9. Confirm to save

A detailed calibration creation guide will be added in a future update.

Community contributions welcome! If you create calibration data for a new track, consider submitting it as a Pull Request to the releases repository in the community/calibration/ folder. Your contribution helps all users.

11. Output Files Reference

All output goes to your telemetry_logs/ folder.

LocationContents
telemetry_logs/Per-session _laps.csv and _turns.csv files
telemetry_logs/raw_telemetry/Raw binary capture files (.bin)
telemetry_logs/analysis/Analysis chart sets (one folder per analysis run)
telemetry_logs/llm_analysis_requests/LLM analysis ZIP bundles
telemetry_logs/charts/optional, high-resolution lap telemetry plots
telemetry_logs/calibration/Track calibration JSON files

File naming follows the pattern: {YYYY}_{MM}_{Mon}_{DD}_{Track}_{Session} (e.g., 2026_01_Jan_30_AbuDhabi_Race_laps.csv).

For complete CSV field documentation, see CSV_OUTPUT_FORMAT.md.
